# 글로컬30 R&D Hub v3 — 개발 핸드오프 인덱스

이 문서는 현재까지 정리된 핵심 설계 문서를 한 번에 안내하기 위한 인덱스다.
다른 AI 에이전트 또는 개발자가 작업을 시작할 때 가장 먼저 이 문서를 읽고 필요한 문서로 이동하면 된다.

- 파일: `db-schema-spec.md`

### 5. 페이지별 구현 체크리스트
- 파일: `implementation-checklist.md`
- 역할: 화면/기능 단위 구현 순서 및 누락 방지 체크리스트

### 6. API 초안 명세
- 파일: `api-spec-draft.md`
- 역할: 프론트/백엔드 인터페이스 초안

## 3. 현재 기준으로 이미 확정된 핵심 원칙

- 프로젝트는 태그가 아니라 `project_id`로 직접 연결한다.
- 데일리는 자유 텍스트 에디터를 사용하되, 블록 단위로 구조화한다.
- 블록마다 프로젝트, 섹션, 공개범위를 지정할 수 있다.
- 프로젝트 없는 블록도 허용한다.
- 교수 권한과 프로젝트 권한은 분리한다.
- 외부업체는 참여 프로젝트 범위 안에서는 넓게 보되, 프로젝트 외 개인 정보는 차단한다.
- 태스크는 공동 할당 가능해야 한다.
- 캘린더는 단순 일정표가 아니라 업무 상태를 파악하는 화면이어야 한다.

---

## 4. 아직 남아 있는 정책 결정 사항

- `internal` 공개범위를 전체 내부로 할지, 랩 단위로 할지
- 프로젝트 생성 권한을 admin 외 누구까지 허용할지
- external의 일정 생성/수정 권한 범위
- Rocket.Chat 채널 자동 생성 정책
- Google Calendar 연동 방식

- 데일리 에디터 및 블록 편집 UI
- 캘린더 화면
