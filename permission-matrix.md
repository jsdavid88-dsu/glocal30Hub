# 글로컬30 R&D Hub v3 — 권한 매트릭스

이 문서는 사용자 역할과 관계에 따라 어떤 데이터를 보고, 만들고, 수정할 수 있는지 정의한다.
이 문서의 목적은 백엔드 권한 체크와 프론트 UI 노출 기준을 명확히 하는 것이다.

---

## 1. 기본 개념

권한 판단은 아래 두 축을 함께 사용한다.

- 전역 역할(role)
  - `admin`
  - `professor`
  - `student`
  - `external`
- 관계(relation)
  - `advisor_of`
  - `project_member`
  - `project_lead`
  - `project_manager`

즉, 교수라고 해서 모든 프로젝트를 보는 것이 아니고,
프로젝트 멤버라고 해서 학생 개인의 비공개 기록을 보는 것도 아니다.

---

## 2. 전역 역할별 기본 권한

### 2.1 admin
- 전체 사용자 열람/수정 가능
- 전체 프로젝트 열람/수정 가능
- 전체 데일리/블록/댓글 열람 가능
- 전체 태스크 열람/수정 가능
- 전체 출결/일정/리포트 열람 가능
- 역할 부여 가능
- 지도교수 관계 설정 가능
- 프로젝트 권한 조정 가능

### 2.2 professor
- 자기 지도학생 데이터 열람 가능
- 자신이 참여 중인 프로젝트 데이터 열람 가능
- 자신이 프로젝트 관리자/리드면 해당 프로젝트 관리 가능
- 프로젝트에 참여하지 않은 경우 그 프로젝트 전체 데이터는 기본 열람 불가

### 2.3 student
- 자기 데이터 열람/수정 가능
- 참여 중인 프로젝트 데이터 열람 가능
- 프로젝트 권한이 있으면 프로젝트 관련 생성/수정 가능

### 2.4 external
- 참여 중인 프로젝트 데이터 열람 가능
- 프로젝트 관련 댓글/태스크/일정 접근 가능
- 프로젝트 외 학생 개인 데이터 열람 불가

---

## 3. 데이터 유형별 권한 규칙

## 3.1 사용자 정보

### 조회
- admin: 전체 조회 가능
- professor: 자기 지도학생 조회 가능
- student: 자기 정보 조회 가능
- external: 자기 정보 조회 가능

### 수정
- admin: 전체 수정 가능
- user 본인: 자기 프로필 일부 수정 가능
- professor: 학생 역할/회사 정보 등 전역 계정 정보 수정 불가

---

## 3.2 지도교수 관계

### 조회
- admin: 전체 조회 가능
- professor: 자신과 연결된 학생 관계 조회 가능
- student: 자신의 지도교수 관계 조회 가능

### 수정
- admin만 가능

---

## 3.3 프로젝트

### 조회
- admin: 전체 조회 가능
- project member: 해당 프로젝트 조회 가능
- professor: 자기 지도학생이 참여한 프로젝트라고 해도, 교수 본인이 참여 중이 아니면 프로젝트 전체 조회는 기본 불가

### 생성
- admin 가능
- project 생성 권한이 있는 내부 사용자 가능 여부는 운영 정책에 따라 허용 가능
- 기본 권장: admin + 내부 권한자

### 수정
- admin 가능
- 해당 프로젝트 `manager` 또는 `lead` 가능

---

## 3.4 프로젝트 멤버 관리

### 조회
- admin 가능
- 해당 프로젝트 참여자 가능

### 추가/제거/역할변경
- admin 가능
- 해당 프로젝트 `manager` 또는 `lead` 가능

---

## 3.5 태스크

### 조회
- admin 가능
- 해당 프로젝트 참여자 가능

### 생성
- admin 가능
- 해당 프로젝트 `manager` 또는 `lead` 가능

### 수정
- admin 가능
- 해당 프로젝트 `manager` 또는 `lead` 가능
- 담당자는 상태 업데이트 가능하도록 허용 가능

### 배정
- admin 가능
- 해당 프로젝트 `manager` 또는 `lead` 가능

---

## 3.6 데일리 로그

데일리는 `daily_logs`와 `daily_blocks`로 나뉜다.
실질적 권한은 블록 기준으로 판단한다.

### daily_logs 조회
- admin: 전체 가능
- 작성자 본인: 가능
- 지도교수: 해당 학생 데일리 헤더는 볼 수 있으나, 실제 표시 내용은 블록 권한으로 제한하는 것이 안전
- 프로젝트 참여자: 프로젝트 연결 블록이 포함된 경우만 실제 내용 접근 가능

### daily_logs 수정
- 작성자 본인 가능
- admin 가능
- 시간 제한 없음

---

## 3.7 데일리 블록

### visibility = private
조회 가능:
- 작성자 본인
- admin

수정 가능:
- 작성자 본인
- admin

### visibility = advisor
조회 가능:
- 작성자 본인
- admin
- 해당 작성자의 지도교수

수정 가능:
- 작성자 본인
- admin

### visibility = internal
조회 가능:
- 작성자 본인
- admin
- 내부 사용자 범위

주의:
- internal의 범위는 추후 `전체 내부` 또는 `랩 단위` 중 하나로 정책 확정 필요

### visibility = project
조회 가능:
- 작성자 본인
- admin
- 연결된 프로젝트 참여자

수정 가능:
- 작성자 본인
- admin

---

## 3.8 댓글

댓글은 블록에 달린다.
댓글 작성 및 조회 권한은 원칙적으로 원본 블록 접근 권한을 따른다.

### 조회
- 해당 블록 조회 가능자만 가능

### 작성
- 해당 블록 조회 가능자 중 댓글 작성 허용 대상
- external도 프로젝트 블록에는 댓글 작성 가능하게 두는 것을 기본 권장

### 수정/삭제
- 작성자 본인
- admin

---

## 3.9 출결

### 조회
- admin: 전체 조회 가능
- user 본인: 자기 조회 가능
- professor: 자기 지도학생 조회 가능
- external: 자기 조회만 가능

### 수정
- admin 가능
- 본인 수정 허용 여부는 운영 정책에 따름
- 기본 권장: 본인 체크인/체크아웃 가능, 과거 수정은 admin 또는 승인 절차 필요

---

## 3.10 일정(Event)

### 조회
- admin: 전체 조회 가능
- 작성자 본인: 가능
- visibility에 따라 추가 공개
- project visibility: 프로젝트 참여자 조회 가능
- advisor visibility: 지도교수 조회 가능
- private visibility: 본인 + admin만 가능

### 생성
- admin 가능
- 일반 사용자는 자신이 접근 가능한 범위에서 생성 가능
- 프로젝트 일정은 해당 프로젝트 `manager` 또는 `lead` 권장

### 수정
- admin 가능
- 생성자 가능
- 프로젝트 일정은 `manager` 또는 `lead` 가능

---

## 3.11 태그

### 조회
- 접근 가능한 블록이나 프로젝트에 연결된 태그만 실사용상 노출
- 전역 태그는 내부 사용자에게 노출 가능

### 생성/수정
- 전역 태그: admin 권장
- 프로젝트 전용 태그: admin 또는 해당 프로젝트 manager/lead 권장

---

## 3.12 리포트

### 조회
- admin: 전체 가능
- professor: 자기 지도학생 관련, 자신이 참여한 프로젝트 관련 리포트 조회 가능
- student: 자기 관련 리포트, 참여 프로젝트 리포트 조회 가능
- external: 참여 프로젝트 범위 리포트 조회 가능

### 생성
- 초기에는 admin 또는 시스템 생성
- 이후 프로젝트 manager/lead 수동 생성 허용 가능

## 4. UI 노출 규칙 메모

- 권한 없는 대상은 버튼 자체를 숨긴다.
- 하지만 프론트 숨김만으로 끝내지 말고 백엔드에서도 반드시 검증한다.
- 블록 단위 권한이 있으므로, 데일리 상세 화면에서도 블록별로 표시/비표시가 갈릴 수 있다.
- 프로젝트 목록, 태스크 목록, 일정 목록은 사용자 권한에 맞는 결과만 내려줘야 한다.

---

## 5. 구현 우선순위

처음 구현할 때 가장 먼저 강제할 권한은 아래다.

1. 프로젝트 멤버만 프로젝트 데이터 열람 가능
2. 프로젝트 없는 블록은 visibility 기준 열람
3. 지도교수는 advisor 블록과 지도학생 상태 조회 가능
4. external은 프로젝트 외 개인 데이터 접근 불가
5. admin은 전체 접근 가능

---

## 6. 미확정 항목

- `internal` 범위를 전체 내부로 할지 랩 단위로 할지
- 학생이 과거 출결 수정 가능한지
- 프로젝트 생성 권한을 누가 가지는지
- external의 프로젝트 일정 생성 권한 허용 여부
