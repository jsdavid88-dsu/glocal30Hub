**작성일:** 2026-03-11
**목적:** 기존 v3-plan.md와 implementation-checklist.md의 문제점을 반영한 수정 설계

## 1. 핵심 문제 진단

### 1.1 역할별 행동 구분 부재
- 데일리 작성 주체 미정의 (교수/외부업체도 쓸 수 있는 구조)
- 출결을 모든 역할에 노출
- 멤버 페이지가 목록만 보여주는 무의미한 페이지

### 1.2 위클리 흐름 부재
- 데일리만 있고 주간 정리 + 배정 흐름이 없음
- 위클리 → 주간보고서 자동 생성 경로 없음

### 1.3 태스크/논문 배정 흐름 부재
- 교수가 학생에게 일을 배정하는 UX 없음
- 배정된 태스크가 학생 데일리 입력창에 연동되는 구조 없음

### 1.4 권한 세분화 부족
- 지도교수가 보는 학생 정보 vs 프로젝트 리더가 보는 멤버 정보 구분 없음

### 1.5 UI 일관성
- Projects 페이지가 다른 페이지와 디자인 톤이 다름 (빽빽하고 지저분)

### 2.1 태스크 배정 권한 상세

## 3. 핵심 워크플로우

### 3.1 주간 사이클

```
[위클리 회의 (기본 월요일, 설정 가능)]
  교수: 지난주 학생별 데일리/태스크 리뷰
  교수: 이번 주 태스크 배정 (논문 URL, 가이드, 설명 첨부)
  학생: 배정된 태스크 확인
  → 위클리 기록 저장

[화~금 데일리]
  학생: 데일리 입력창에 "배정된 태스크" 목록이 보임
  학생: 각 태스크 맥락 안에서 진행상황 기록
  교수: 학생 데일리 열람 + 블록별 댓글

[주간 마감 시점]
  주간보고서 자동 생성 (학생별 태스크 진행 + 데일리 블록 집계)
  미완료 태스크 → 다음 주 위클리로 자동 이월 (동일 태스크, week 연결만 변경)
```

### 3.2 태스크 배정 흐름

```
교수/프로젝트리더가 위클리 or 대시보드에서 배정
  → 태스크 생성: 제목, 설명, 참고 URL, 접근법 가이드
  → 학생에게 할당
  → 학생 데일리 입력창에 "이번 주 배정 태스크" 섹션으로 노출
  → 학생이 데일리에서 해당 태스크 아래 진행 기록
  → 완료/진행중/블로킹 상태 추적
```

### 3.3 데일리 입력 흐름 (학생 관점)

데일리 에디터는 **태스크 중심** 구조를 기본으로 한다.
배정된 태스크 목록이 먼저 보이고, 각 태스크 아래에 진행상황을 기록한다.
태스크와 무관한 내용은 "기타 메모" 영역에 자유 입력한다.
각 블록에는 공개 범위(private/advisor/internal/project)와 태그를 붙일 수 있다.

```
데일리 입력창 진입
  ┌─────────────────────────────┐
  │ 📋 이번 주 배정 태스크       │
  │ ├─ [진행중] GAN 논문 리뷰    │
  │ │   참고: arxiv.org/...     │
  │ │   가이드: "Section 3 중심" │
  │ │   → [여기에 오늘 진행 기록] │
  │ ├─ [새로] 모델 A 벤치마크    │
  │ │   → [여기에 오늘 진행 기록] │
  │ └─ [완료] 데이터셋 전처리    │
  │                             │
  │ 📝 기타 메모                 │
  │   [자유 입력 영역]           │
  │   공개범위: [advisor ▼]     │
  │   태그: [#연구 #아이디어]    │
  └─────────────────────────────┘
```

### 3.4 프로젝트별 그룹핑

위클리, 데일리 피드에서 학생/데이터를 프로젝트별로 묶어서 표시한다.

**위클리 (교수 뷰):**
- 학생 요약 테이블: "전체 보기" / "프로젝트별 보기" 토글
- 프로젝트별 보기 시 프로젝트 헤더 아래 해당 학생들 그룹핑
- 프로젝트별 aggregate 통계 (완료/진행중/미시작 합산)
- 태스크 배정 시 프로젝트 필터로 해당 프로젝트 학생만 드롭 존 표시
- 프로젝트 미배정 학생은 "프로젝트 없음" 그룹으로

**데일리 피드:**
- "시간순" / "프로젝트별" 보기 토글
- 프로젝트별 보기 시 해당 프로젝트 데일리 블록만 묶어서 표시
- 프로젝트와 무관한 블록은 "프로젝트 없음" 섹션

**DB 설계 영향:**
- DailyBlock에 project_id FK 유지 (기존 스키마 대로)
- WeeklySession 테이블 추가 필요: id, week_start, week_end, created_by, meeting_notes
- Task에 week_id FK 추가 (위클리 세션 연결)
- 주간보고서 집계 시 project_id 기준 GROUP BY

### 3.5 주간보고서 자동 생성 메커니즘

주간보고서는 아래 데이터를 집계하여 생성한다:
- 해당 주에 배정된 태스크 목록 + 최종 상태 (완료/진행중/미시작/블로킹)
- 각 태스크에 연결된 데일리 블록들 (학생이 매일 기록한 진행상황)
- 태스크 상태 변경 이력
- 미완료 태스크 → "다음 주 이월" 표시 (동일 태스크 객체, `week_id` 필드만 갱신)

### 3.6 태스크 이월 메커니즘

- 태스크는 주 단위로 `week_id`(위클리 세션)에 연결된다
- 주간 마감 시, status가 `done`이 아닌 태스크는 자동으로 다음 주 `week_id`로 갱신
- 태스크 객체 자체는 복제하지 않음 (하나의 태스크가 여러 주에 걸쳐 추적됨)
- 이월 이력은 태스크의 `week_history` 또는 audit log로 추적

## 4. 페이지 구조 (수정)

### 4.1 대시보드 `/`

**교수 뷰:**
- 내 학생들 현황 (지도학생 + 프로젝트 학생 통합)
- 오늘 데일리 제출 현황 (제출/미제출 한눈에)
- 학생 출결 현황 (오늘 출근한 학생)
- 이번 주 배정 태스크 현황 (완료/진행중/미시작)
- 최근 이슈 블록
- 다가오는 마일스톤/일정

**학생 뷰:**
- 오늘 배정된 태스크 목록 (상세정보 포함)
- 출근/퇴근 버튼
- 데일리 작성 바로가기
- 이번 주 진행률
- 오늘 일정

**외부업체 뷰:**
- 참여 프로젝트 현황만
- 프로젝트별 최근 이슈/태스크

### 4.2 데일리 (학생 전용 작성, 전원 열람)

**`/daily/write` — 학생만 접근**
- 배정된 태스크 목록이 상단에 표시 (참고 URL, 가이드 포함)
- 각 태스크 아래 진행상황 입력 영역
- 기타 메모 자유 입력 (태스크 무관 내용)
- 각 블록에 공개 범위 지정: private / advisor / internal / project
- 태그 지정 가능
- 임시저장 (localStorage + 서버 draft) + 최종 저장

**`/daily/feed` — 역할별 필터링**
- 교수: 지도학생 데일리 열람 + 학생 필터 + 블록별 댓글
- 학생: 본인 + 프로젝트 공유
- 외부업체: 프로젝트 공개 블록만
- 필터: 날짜, 작성자, 프로젝트, 태그, 이슈만 보기

**`/daily/:id` — 상세 보기**
- 블록별 표시 + 연결된 태스크 정보
- 블록별 댓글 (교수 피드백)
- 첨부 파일 보기

### 4.3 위클리 (별도 페이지)

**`/weekly` — 주간 회의 + 배정 도구**

교수 뷰:
- 지난주 학생별 데일리/태스크 요약 자동 집계
- 학생별 완료/진행중/미시작 태스크 현황
- 이번 주 태스크 배정 UI
  - 학생 선택 → 태스크 생성 (제목, 설명, URL, 가이드)
  - 기존 진행중 태스크 이월 확인
- 논문/SOTA 배정
- 위클리 회의록 작성 영역

학생 뷰:
- 지난주 내 활동 요약 (자동 집계)
- 이번 주 배정된 태스크 확인 (상세정보 포함)
- 이번 주 계획 작성

**`/weekly/:id` — 주간보고서**
- 자동 생성: 학생별 배정 태스크 + 진행상황 + 결과
- 미완료 항목 → 다음 주 이월 표시
- 교수 코멘트

### 4.4 프로젝트

**`/projects` — 프로젝트 목록 (UI 정리)**
- 현재 카드형 → 여유있는 깔끔한 리스트형으로 변경
- 다른 페이지와 디자인 톤 통일
- 상태 필터, 검색

**`/projects/:id` — 프로젝트 상세 (통합 뷰)**
- 기본 정보 + 멤버 + 태스크 + 일정/마일스톤을 탭으로 구성
- 프로젝트 리더/매니저: 마일스톤 생성/관리, 태스크 배정
- 태스크 칸반 보드

### 4.5 학생 관리 (기존 `/members` 대체)

**`/students` — 교수 전용**

지도학생 섹션:
- 전체 개인 정보 (관심분야, 강점, 연구 분야)
- 데일리 제출 현황, 출결 현황
- 현재 참여 프로젝트
- 최근 배정 태스크 + 진행상황
- 클릭 → 학생 상세 페이지

프로젝트 멤버 섹션:
- 프로젝트 관련 정보만 (태스크 현황, 프로젝트 데일리, 역할)
- 개인 정보는 제한적 (관심분야 등 비공개)

**`/students/:id` — 학생 상세 (교수 뷰)**
- 프로필 + 관심분야 + 참여 프로젝트
- 데일리 히스토리
- 태스크 히스토리
- 출결 캘린더
- 주간 리포트 연결

### 4.6 출결

**`/attendance` — 학생만 체크인/아웃**
- 체크인/아웃 버튼
- 본인 출결 기록
- 교수는 별도 페이지 불필요 → 대시보드 + 학생 상세에서 확인

### 4.7 캘린더

**`/calendar`**
- 교수: 학생 일정 + 프로젝트 마일스톤 + 위클리 일정
- 학생: 개인 일정 + 프로젝트 일정 + 수업
- 외부업체: 프로젝트 일정만

### 4.8 프로필

**`/profile`**
- 본인 프로필 편집 (이름, 프로필 이미지, 관심분야, 연구분야)
- 소속 정보, 지도교수 표시
- 계정 설정

### 4.9 관리자

**`/admin/users`** — 사용자 목록, 역할 변경, 상태 변경, 지도교수 지정
**`/admin/projects`** — 프로젝트 생성/수정, 멤버 관리
**`/admin/tags`** — 전역 태그 관리
**`/admin/audit`** — 감사 로그 조회

### 4.10 SOTA/논문 (Phase 4)

**`/sota`**
- 논문 목록 (향후 자동 수집)
- 교수: 논문 추가 + 학생 배정
- 학생: 배정된 논문 목록 + 리뷰 제출

### 4.11 리포트

**`/reports`** — 주간보고서 목록 + 기간별/프로젝트별 필터
**`/reports/:id`** — 리포트 상세 (자동 생성 + 교수 코멘트)

## 5. 권한 세분화

### 5.1 관리자 (admin)
- 모든 데이터 열람/수정
- 사용자 역할 변경, 지도교수 관계 설정
- 프로젝트 생성/삭제, 권한 설정
- 감사 로그 열람

### 5.2 지도교수 → 지도학생
- 개인 데일리 전체 열람 (advisor 이상)
- 출결 기록 전체 열람
- 관심분야, 개인 프로필 열람
- 태스크/논문 배정 가능
- private 블록은 열람 불가

### 5.3 프로젝트 리더/매니저 → 프로젝트 멤버
- 프로젝트 관련 데일리만 열람 (project 범위)
- 프로젝트 태스크 배정/관리
- 마일스톤 생성/관리
- 개인 출결, 개인 데일리는 열람 불가
- 프로젝트 역할/기여만 확인

### 5.4 학생 → 다른 학생
- 같은 프로젝트 내 project 범위 블록만
- 개인 정보 열람 불가

### 5.5 외부업체
- 프로젝트 공개(project) 블록만
- advisor/private/internal 접근 불가
- 출결, 개인 데일리 접근 불가
- 프로젝트 멤버 이메일 등 개인정보 비공개

## 6. 수정된 Phase 계획

### Phase 1 (기반)
- Google OAuth 인증
- 사용자/권한 모델 (admin/professor/student/external + 역할별 행동 구분)
- 지도교수 관계
- 프로젝트 + 멤버 구조 (리더/매니저/멤버/뷰어)
- DB 마이그레이션
- 프론트엔드 역할별 라우팅 + 권한 가드
- 프로필 페이지
- 관리자 기본 페이지 (사용자 관리)

### Phase 2a (데일리 + 태스크)
- 데일리 작성 (학생 전용, 태스크 중심 에디터)
- 태스크 생성/배정 (교수/프로젝트리더 → 학생)
- 배정된 태스크 → 데일리 입력창 연동
- 블록별 댓글
- 태그 시스템 (전역 + 프로젝트)
- 첨부 파일 (링크 + 소형 파일 업로드)

### Phase 2b (위클리 + 보고서)
- 위클리 페이지 (회의 + 배정 도구)
- 주간보고서 자동 생성 (태스크 상태 + 데일리 블록 집계)
- 미완료 태스크 자동 이월
- 감사 로그 기본 구조

### Phase 3 (관리/추적)
- 출결 (학생 체크인/아웃 + 교수 대시보드 조회)
- 학생 관리 페이지 (교수 전용, 지도학생+프로젝트 멤버 뷰)
- 캘린더/일정 + 마일스톤
- 인앱 알림
- 리포트 목록/상세 페이지

### Phase 4 (고도화)
- SOTA/논문 배정 (수동 → 향후 자동 수집)
- 예산 관리 (기본)
- LLM 요약 (데일리/위클리 자동 요약)
- Google Calendar 연동
- Rocket.Chat 연동

## 7. 기존 플랜에서 명시적으로 변경/제거한 항목

| 항목 | 변경 내용 | 이유 |
|------|-----------|------|
| `/members` 페이지 | `/students`로 대체 (교수 전용) | 단순 목록은 무의미, 교수가 학생 상태를 깊이 볼 수 있어야 함 |
| 데일리 섹션 구조 | yesterday/today/issue/misc → 태스크 중심 + 기타 메모 | 배정된 태스크 맥락에서 기록하는 것이 핵심 |
| 데일리 작성 권한 | 전원 → 학생만 | 교수/외부업체는 작성 불필요, 댓글로 충분 |
| 출결 페이지 접근 | 전원 → 학생만 | 교수/외부업체 체크인 불필요, 교수는 대시보드에서 조회 |
| Phase 2 분할 | Phase 2 → 2a + 2b | 데일리+태스크와 위클리+보고서를 단계적으로 구현 |
| 위클리 추가 | 없음 → `/weekly` 별도 페이지 | 주간 회의 + 배정 + 보고서 생성이 핵심 워크플로우 |
